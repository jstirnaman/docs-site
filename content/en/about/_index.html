My goals

  1. Practice coding, managing content with Hugo, automating content pipelines, and deploying.
  2. Practice technical writing.
  3. Use OSS and automation to generate templated Markdown documents from an API specification.
  4. Develop Markdown templates that structure API endpoint documentation in a user-friendly format.
  5. Transform the generated Markdown to populate the template(s).
  6. Use Hugo, the popular static site generator, and open-source themes to render the templated documentation.
  7. Enhance the documentation with the following features: code samples, "try-it", theme parameters.

About the author

I'm a technical writer, documentation engineer, and developer. I fell in love with metadata (for biodiversity sampling) and the Web in the late 1990's. After earning my Master's in Library and Information Science, I managed library systems, web sites, and content repositories at Missouri State Library and University of Kansas Medical Center.

Out of a hunger to solve problems, a fear of tedium, and a passion for information sharing, I continued learning agile, programming, and DevOps. Using Ruby on Rails, Solr, and API integrations, I worked on projects to capture showcase medical school faculty expertise and publishing. I had the pleasure of learning from and working with great developers who taught me the value and practice of automated tests and I learned how to automate processes I'd never considered.

After working as a full-stack engineer, frontend engineer, and technical lead in a number of organizations, I was reminded that I enjoy:

  • Helping people learn.
  • Sharing information.
  • Applying automation to improve quality and to free people up for more creative work.
  • Writing and editing.

In 2021, I decided to take a risk on technical writing. Thankfully, the InfluxData Docs Team took a risk on me. It was a great team and the perfect mix of writing, automating, playing with APIs, and helping developers. I also volunteered as a technical writer with the Ethereum Foundation, writing guides for the Semaphore protocol and smart contracts.

After InfluxData announced layoffs in November 2022, I started this site as a proving ground for ideas that I'd planned to implement there.

When I'm not spending my time on docs and code, I'm playing with my family, cooking, cycling, or enjoying music. Don't be surprised if some of those themes trickle in here.
